## 4. 下一步可继续扩展点

### 4.1 优先级 P1

- 新建数据库名称非法字符、前后空白、重复数据库名校验
- 新建测点标签更多异常输入组合，例如多段非法 `key=value`
- 导入测点模板内容校验更细粒度断言，例如列类型、示例值
- 列表页“最新值 / 最新值时间”在有实时数据写入场景下的动态刷新断言

### 4.2 优先级 P2

- 测点列表更多复杂筛选组合，例如“描述筛选 + 列过滤 + 刷新 + 导出”串联
- 数据模型页更多可视化交互，例如深层节点多次展开、分页后回到目标节点
- root 主节点与普通数据库节点的右键菜单差异校验
- 数据查询页、运行趋势页、历史趋势页与测点管理之间的跨模块回跳链路

### 4.3 优先级 P3

- 多浏览器项目回归，例如 WebKit / Firefox
- 失败自动截图、视频与报告的二次归档策略
- 将测点管理模块继续拆成更细的批次回归脚本或报告入口

## 5. 维护建议

- 后续页面结构变化时，优先维护 `tests/e2e/pages/measurement-management-page.ts`
- 若真实环境不稳定，优先保留“前置数据走 API、页面只做核心交互验证”的策略
- 新增用例时保持“中文编号标题 + 注释 + XMind 同步”三者一致

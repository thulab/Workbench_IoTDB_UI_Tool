# IoTDB Workbench UI 自动化 Xmind 测试用例树

本文档用于沉淀 UI 自动化测试点，采用 Xmind 树形展开思路整理。当前已收口登录页、实例管理、首页 Dashboard、查询页四类模块，后续可继续扩展测点管理等模块。

## 1. 登录页

### 1.1 页面加载与基础展示

#### 1.1.1 进入登录页并加载 localhost 实例

- 前置条件
  - Workbench 已启动并监听 `http://127.0.0.1:9190`
  - IoTDB 已启动
  - 系统中存在实例名称为 `localhost` 的可用连接
  - 页面语言为简体中文
- 操作步骤
  - 打开登录页
  - 等待登录表单渲染完成
  - 在“请选择所要选择的实例”下拉框中选择 `localhost`
- 预期结果
  - 登录页加载成功
  - 登录表单展示实例、用户名、密码、模型、登录按钮
  - 默认模型为“树模型”
  - 选择 `localhost` 后，用户名自动回填为 `root`

### 1.2 表单必填校验

#### 1.2.1 连接实例为空时点击登录

- 前置条件
  - 已进入登录页
  - “请选择所要选择的实例”为空
  - 用户名与密码保持默认空值
- 操作步骤
  - 直接点击【登录】
- 预期结果
  - 实例选择框下方显示红字提示：`请选择连接实例`
  - 页面停留在登录页
  - 不跳转到首页

#### 1.2.2 用户名为空时点击登录

- 前置条件
  - 已进入登录页
  - 已选择实例 `localhost`
  - 用户名为空
  - 密码填写为有效密码 `TimechoDB@2021`
- 操作步骤
  - 点击【登录】
- 预期结果
  - 用户名输入框下方显示红字提示：`用户名不能为空`
  - 页面停留在登录页
  - 不跳转到首页

#### 1.2.3 密码为空时点击登录

- 前置条件
  - 已进入登录页
  - 已选择实例 `localhost`
  - 用户名填写为 `root`
  - 密码为空
- 操作步骤
  - 点击【登录】
- 预期结果
  - 密码输入框下方显示红字提示：`密码不能为空`
  - 页面停留在登录页
  - 不跳转到首页

### 1.3 登录结果校验

#### 1.3.1 用户名和密码不匹配时登录失败

- 前置条件
  - 已进入登录页
  - 已选择实例 `localhost`
  - 用户名填写为 `root`
  - 密码填写为错误值
- 操作步骤
  - 点击【登录】
- 预期结果
  - 页面出现错误浮层提示：`用户名或密码错误，请修改后重新操作`
  - 页面停留在登录页
  - 不展示首页左侧菜单

#### 1.3.2 用户名和密码正确时登录成功

- 前置条件
  - 已进入登录页
  - 已选择实例 `localhost`
  - 用户名填写为 `root`
  - 密码填写为 `TimechoDB@2021`
- 操作步骤
  - 点击【登录】
- 预期结果
  - 成功跳转到首页 `Dashboard`
  - 首页展示当前实例名 `localhost`
  - 首页展示连接地址 `127.0.0.1:6667`
  - 页面可见“首页”“系统信息”等核心区域

### 1.4 登录后退出与清理

#### 1.4.1 登录成功后退出登录并删除 localhost 实例

- 前置条件
  - 已使用 `localhost` 成功登录
- 操作步骤
  - 进入首页后，鼠标悬停右上角用户名
  - 点击【退出登录】
  - 返回登录页后点击【编辑】
  - 进入实例管理界面删除 `localhost`
  - 关闭实例管理弹窗
- 预期结果
  - 成功退出到登录页
  - `localhost` 实例被删除
  - 登录页仍可正常展示

## 2. 实例管理

### 2.1 实例新建

#### 2.1.1 完成单机实例新建流程

- 前置条件
  - 已进入登录页
  - 点击【编辑】进入实例管理弹窗
- 操作步骤
  - 点击新增实例
  - 保持连接类型为“单机”
  - 输入实例名称、主机、端口、用户名、密码
  - 点击【测试】
  - 点击【保存】
- 预期结果
  - 测试连接成功提示出现
  - 实例列表新增对应实例
  - 保存后实例详情可再次打开查看

#### 2.1.2 保存空白表单时显示必填校验

- 前置条件
  - 已进入实例管理弹窗
  - 已打开新增实例表单
- 操作步骤
  - 不输入任何信息
  - 点击【保存】
- 预期结果
  - 表单保留在当前弹窗
  - 实例名称、主机、端口、用户名等必填项出现红字校验

### 2.2 表单操作

#### 2.2.1 重置新建表单后恢复默认值

- 前置条件
  - 已进入实例管理弹窗
  - 已打开新增实例表单并填写部分内容
- 操作步骤
  - 点击【重置】
- 预期结果
  - 连接类型恢复为“单机”
  - 默认模型恢复为“树模型”
  - 所有输入框恢复为空值

#### 2.2.2 切换实例时显示未保存确认

- 前置条件
  - 已进入实例管理弹窗
  - 已创建至少 1 个已保存实例
  - 当前存在未保存草稿
- 操作步骤
  - 点击另一条实例记录
  - 先点击“继续编辑”
  - 再次切换并点击“放弃修改”
- 预期结果
  - 第一次切换后保留当前草稿内容
  - 第二次切换后草稿被放弃
  - 表单回到已保存实例内容

### 2.3 编辑与删除

#### 2.3.1 编辑已保存实例并持久化更新

- 前置条件
  - 已存在可编辑实例
- 操作步骤
  - 打开目标实例详情
  - 修改实例名称、主机、端口
  - 点击【保存】
  - 刷新实例列表后重新打开该实例
- 预期结果
  - 保存成功提示出现
  - 列表展示更新后的实例名称
  - 表单中可查看更新后的主机与端口

#### 2.3.2 删除已保存实例

- 前置条件
  - 已存在可删除实例
- 操作步骤
  - 在实例列表点击删除
  - 在确认弹层点击确认
- 预期结果
  - 删除成功提示出现
  - 目标实例从列表中消失

### 2.4 连接测试与附加信息

#### 2.4.1 使用错误密码测试连接失败

- 前置条件
  - 已打开新增实例表单
- 操作步骤
  - 输入有效主机、端口、用户名
  - 输入错误密码
  - 点击【测试】
- 预期结果
  - 页面出现失败提示
  - 不出现成功提示

#### 2.4.2 localhost 编辑态展示 Prometheus 地址

- 前置条件
  - 已存在 `localhost` 实例
- 操作步骤
  - 打开 `localhost` 实例详情
- 预期结果
  - 表单中展示 Prometheus 地址 `127.0.0.1:9090`

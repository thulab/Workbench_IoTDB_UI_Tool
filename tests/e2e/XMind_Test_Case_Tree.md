# IoTDB Workbench UI 自动化 XMind 测试用例树

本文档用于沉淀当前 UI 自动化测试点，采用 XMind 树形展开方式整理，覆盖树模型下已落地的登录、实例管理、首页 Dashboard、查询模块、SQL操作模块、视图页面、测点管理等模块。

## 1. 登录页

### 1.1 页面加载与基础展示

#### 1.1.1 进入登录页并加载 `localhost` 实例

- 前置条件
  - Workbench 已启动并监听 `http://127.0.0.1:9190`
  - IoTDB 已启动
  - 系统中存在实例名称为 `localhost` 的可用连接
- 操作步骤
  - 打开登录页
  - 等待登录表单渲染完成
  - 在实例下拉框中选择 `localhost`
- 预期结果
  - 登录页加载成功
  - 页面展示实例、用户名、密码、登录按钮
  - 默认模型为“树模型”
  - 选择 `localhost` 后用户名可自动回填为 `root`

### 1.2 表单必填校验

#### 1.2.1 未选择连接实例时点击登录

- 前置条件
  - 已进入登录页
- 操作步骤
  - 不选择实例，直接点击【登录】
- 预期结果
  - 红字提示：`请选择连接实例`
  - 页面停留在登录页

#### 1.2.2 用户名为空时点击登录

- 前置条件
  - 已进入登录页
  - 已选择 `localhost`
- 操作步骤
  - 用户名留空
  - 输入有效密码后点击【登录】
- 预期结果
  - 红字提示：`用户名不能为空`
  - 页面停留在登录页

#### 1.2.3 密码为空时点击登录

- 前置条件
  - 已进入登录页
  - 已选择 `localhost`
- 操作步骤
  - 输入用户名 `root`
  - 密码留空并点击【登录】
- 预期结果
  - 红字提示：`密码不能为空`
  - 页面停留在登录页

### 1.3 登录结果校验

#### 1.3.1 用户名和密码不匹配时登录失败

- 前置条件
  - 已进入登录页
  - 已选择 `localhost`
- 操作步骤
  - 输入用户名 `root`
  - 输入错误密码
  - 点击【登录】
- 预期结果
  - 浮层提示：`用户名或密码错误，请修改后重新操作`
  - 页面停留在登录页

#### 1.3.2 用户名和密码正确时登录成功

- 前置条件
  - 已进入登录页
  - 已选择 `localhost`
- 操作步骤
  - 输入用户名 `root`
  - 输入密码 `TimechoDB@2021`
  - 点击【登录】
- 预期结果
  - 成功跳转到 Dashboard
  - 页面展示 `localhost` 实例与 `127.0.0.1:6667`

### 1.4 退出与清理

#### 1.4.1 登录成功后退出登录并清理 `localhost` 实例

- 前置条件
  - 已使用 `localhost` 成功登录
- 操作步骤
  - Hover 右上角用户名
  - 点击【退出登录】
  - 返回登录页后点击【编辑】
  - 在实例管理中删除 `localhost`
- 预期结果
  - 成功退出到登录页
  - `localhost` 实例被清理
